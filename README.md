# wastedata-ca-data

California waste management data pipeline. Downloads raw disposal and material composition data from CalRecycle and the California Department of Finance, transforms it into clean JSON, and publishes it to GitHub Pages for runtime consumption.

This repository serves as the data backend for [wastedata-ca](https://github.com/maxnelson/wastedata) and as a standalone data source for developers, researchers, and data enthusiasts working with California solid waste data.

---

## Published Data Endpoints

All processed data is available via GitHub Pages at no cost. No API key required.

**Base URL:** `https://maxnelson.github.io/wastedata-ca-data`

| Endpoint | Description |
|----------|-------------|
| `/core/jurisdictions.json` | Master registry of all 419 CA jurisdictions with disposal metrics |
| `/core/population.json` | Annual population estimates per jurisdiction (2020–2025) |
| `/core/disposal/by_jurisdiction.json` | Quarterly disposal tonnage per jurisdiction (2019–2025) |
| `/core/disposal/jurisdiction_county_map.json` | Maps each jurisdiction name to its county |
| `/{slug}.json` | Material composition breakdown for a specific jurisdiction |

**Example — fetch Berkeley's characterization data:**
```js
const res = await fetch('https://maxnelson.github.io/wastedata-ca-data/berkeley.json')
const data = await res.json()
// data.commercial.categories["Organic"] → { tons: 25944.9, pct: 40.1 }
```

**Example — fetch all jurisdictions:**
```js
const res = await fetch('https://maxnelson.github.io/wastedata-ca-data/core/jurisdictions.json')
const jurisdictions = await res.json()
// jurisdictions["Berkeley"] → { slug, county, pop2024, perCapita, yoy, ... }
```

A tarball of the four core files is also published as a GitHub Release at tag `data-latest`:
```
https://github.com/maxnelson/wastedata-ca-data/releases/download/data-latest/core-data.tar.gz
```

## Processed Data Reference

### `core/jurisdictions.json`

Master registry. One entry per jurisdiction. 419 jurisdictions total.

```json
{
  "Berkeley": {
    "name": "Berkeley",
    "slug": "berkeley",
    "county": "Alameda",
    "pop2024": 124506,
    "q1Total2024": 22318.5,
    "q1Total2023": 23105.1,
    "yoy": -3.40,
    "perCapita": 3.97,
    "hasCharacterization": true,
    "commercial": {
      "Organic": { "tons": 9127.0, "pct": 40.9 },
      "Paper & Cardboard": { "tons": 5480.0, "pct": 24.6 },
      ...
    },
    "residential": {
      "Organic": { "tons": 5210.0, "pct": 51.2 },
      ...
    }
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Canonical CalRecycle jurisdiction name |
| `slug` | string | URL-safe identifier (e.g. `"san-francisco"`) |
| `county` | string | County this jurisdiction belongs to |
| `pop2024` | number \| null | 2024 population from CA DOF E-4 estimates |
| `q1Total2024` | number | Q1 2024 total disposal in tons |
| `q1Total2023` | number \| null | Q1 2023 total disposal in tons (for YoY) |
| `yoy` | number \| null | Year-over-year % change in disposal (Q1 2024 vs Q1 2023) |
| `perCapita` | number \| null | Q1 2024 disposal in lbs per person per day |
| `hasCharacterization` | boolean | Whether a characterization JSON exists for this jurisdiction |
| `commercial` | object \| absent | Material category breakdown for commercial waste (tons, pct per category) |
| `residential` | object \| absent | Material category breakdown for residential waste (tons, pct per category) |

The eight material categories are: `Organic`, `Paper & Cardboard`, `Plastic`, `Construction & Inerts`, `Metal`, `Mixed Residue`, `Special Waste`, `Glass`.

---

### `core/population.json`

Annual population estimates per jurisdiction, sourced from the CA Department of Finance E-4 report.

```json
{
  "Berkeley": {
    "county": "Alameda",
    "pop": {
      "2020": 117165,
      "2021": 120879,
      "2022": 123143,
      "2023": 124200,
      "2024": 124506,
      "2025": 124800
    }
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `county` | string | County this jurisdiction belongs to |
| `pop` | object | Map of year string → population integer |

Covers incorporated cities and unincorporated county areas. Population is as of January 1 of each year per the DOF E-4 methodology.

---

### `core/disposal/by_jurisdiction.json`

Quarterly solid waste disposal tonnage per jurisdiction, sourced from CalRecycle annual disposal tonnage spreadsheets (2019–2025). 420 jurisdictions, ~10,500 quarterly records total.

```json
{
  "Berkeley": [
    {
      "year": 2024,
      "quarter": 1,
      "landfill": 21814.2,
      "transformation": 0.0,
      "emsw": 504.3,
      "greenADC": 198.5,
      "total": 22318.5
    },
    ...
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `year` | integer | Calendar year |
| `quarter` | integer | Quarter number (1–4) |
| `landfill` | number | Tons disposed at landfill |
| `transformation` | number | Tons processed at transformation facilities (e.g. WTE) |
| `emsw` | number | Exported municipal solid waste (tons) |
| `greenADC` | number | Green material used as alternative daily cover (not counted in total) |
| `total` | number | Total disposed tons (landfill + transformation + emsw) |

`total` is the primary metric used for per-capita calculations. `greenADC` is tracked separately per CalRecycle methodology and excluded from total disposal.

---

### `core/disposal/jurisdiction_county_map.json`

Simple lookup mapping each jurisdiction name to its county. Useful for county-level aggregation.

```json
{
  "Berkeley": "Alameda",
  "Los Angeles": "Los Angeles",
  "Unincorporated Riverside County": "Riverside",
  ...
}
```

420 entries covering all jurisdictions with disposal data.

---

### `characterization/{slug}.json`

Material composition breakdown for a specific jurisdiction, sourced from CalRecycle's Statewide Waste Characterization Study. One file per jurisdiction that has characterization data (~356 files). Served at `https://maxnelson.github.io/wastedata-ca-data/{slug}.json`.

```json
{
  "commercial": {
    "total": 2807.1,
    "categories": {
      "Organic":              { "tons": 1115.0, "pct": 39.72 },
      "Paper & Cardboard":    { "tons":  688.2, "pct": 24.51 },
      "Plastic":              { "tons":  343.0, "pct": 12.22 },
      "Construction & Inerts":{ "tons":  355.5, "pct": 12.66 },
      "Metal":                { "tons":  123.9, "pct":  4.41 },
      "Mixed Residue":        { "tons":   32.0, "pct":  1.14 },
      "Special Waste":        { "tons":   93.6, "pct":  3.33 },
      "Glass":                { "tons":   56.0, "pct":  2.00 }
    }
  },
  "residential": {
    "total": 8180.0,
    "categories": { ... }
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `commercial.total` | number | Total commercial waste characterized (tons) |
| `commercial.categories` | object | Material breakdown with `tons` and `pct` per category |
| `residential.total` | number | Total residential waste characterized (tons) |
| `residential.categories` | object | Material breakdown with `tons` and `pct` per category |

The characterization study is conducted periodically rather than annually, so these percentages reflect the most recent study for each jurisdiction and do not vary by year or quarter.

To look up the slug for a jurisdiction, use the `slug` field in `jurisdictions.json` or apply the same slugify rule: lowercase, strip parentheses, replace non-alphanumeric runs with hyphens, trim leading/trailing hyphens.

## Raw Data

The `data/raw/` directory contains the original source files:

### `data/raw/disposal-tonnage/`
Seven Excel files from CalRecycle's Annual Disposal Tonnage by Jurisdiction report, one per year (2019–2025). Columns: Year, Quarter, Jurisdiction, County, Landfill, Transformation, EMSW, Green Material ADC. Each file covers all reporting California jurisdictions for that calendar year.

### `data/raw/population/`
One Excel file from the CA Department of Finance Population Estimates and Projections program (E-4 series). Covers cities and unincorporated county areas, 2020–2025.

### `data/raw/waste-characterization/`
1,200+ Excel files organized by county and jurisdiction, downloaded from CalRecycle's Waste Characterization Tool. Each jurisdiction has separate files for commercial and residential material streams. Some counties also have business-group breakdowns. File naming convention: `{jurisdiction-slug}_{stream}.xlsx`.

## Tools: Data Extraction

### `tools/data_extraction/fetch-calrecycle-waste-characterization.cjs`

Downloads commercial and residential waste characterization Excel files for all jurisdictions in a given California county from CalRecycle's Waste Characterization Tool.

**Usage:**
```bash
# Install dependencies (one-time)
cd tools/data_extraction && npm install

# Download a single county
node fetch-calrecycle-waste-characterization.cjs "Alameda"

# Download with a longer delay between requests (default: 7 seconds)
node fetch-calrecycle-waste-characterization.cjs "Los Angeles" --delay 10000

# Discover all available county names and IDs
node fetch-calrecycle-waste-characterization.cjs --discover
```

**What it does:**
- Authenticates a session with the CalRecycle Waste Characterization Tool
- Iterates over all jurisdictions in the requested county
- Downloads both commercial (material type streams) and residential (streams) Excel exports
- Saves files to `data/raw/waste-characterization/{county-slug}/{stream}/`
- Rate-limits requests to 7 seconds apart by default to be respectful of public infrastructure
- Resume-safe: skips jurisdictions that already have output files

**Dependencies:** `axios`, `cheerio` (Node.js)

### `tools/data_extraction/download-all-counties.sh`

Batch wrapper that runs `fetch-calrecycle-waste-characterization.cjs` for all 58 California counties in sequence.

**Usage:**
```bash
bash tools/data_extraction/download-all-counties.sh

# Log output to file
bash tools/data_extraction/download-all-counties.sh 2>&1 | tee logs/download.log
```

**What it does:**
- Iterates all 58 counties in a fixed order
- Checks whether a county's output directory already contains files; skips it if so (resume-safe)
- Reports downloaded / skipped / failed counts at the end
- At the default 7-second delay, a full download of all 58 counties takes several hours

## Tools: Data Transformation

All transformation scripts are pure Python 3, stdlib-only. No pandas or third-party dependencies required.

Run from the repository root:
```bash
npm run transform:all
```
This runs all four transform scripts sequentially in the correct order.

Or run individually:
```bash
python3 tools/data_transformation/transform_disposal.py
python3 tools/data_transformation/transform_population.py
python3 tools/data_transformation/transform_characterization.py
python3 tools/data_transformation/build_processed.py
```

---

### `tools/data_transformation/lib.py`

Shared utilities used by all transform scripts.

**`read_sheet(xlsx_path, sheet_number)`**
Reads all rows from a given sheet in an Excel file. Returns a list of rows; each row is a list indexed by 0-based column position. Missing cells are represented as empty strings. Handles sparse rows correctly via the Excel cell reference (`r` attribute) rather than relying on row ordering. Pure stdlib — reads xlsx files as zip archives and parses the XML directly.

**`to_float(s)`**
Converts a cell value string to float, returning `0.0` for empty or unparseable values.

**`slugify(name)`**
Converts a jurisdiction name to a URL-safe slug: lowercase, strip parentheses, replace non-alphanumeric runs with hyphens, trim leading/trailing hyphens. Used consistently across all scripts and the React app to ensure slug lookups match across datasets.

---

### `tools/data_transformation/transform_disposal.py`

**Input:** `data/raw/disposal-tonnage/calrecycle-disposal-{year}.xlsx` (7 files, 2019–2025)

**Output:**
- `data/processed/disposal/by_jurisdiction.json` — quarterly tonnage records keyed by jurisdiction name
- `data/processed/disposal/jurisdiction_county_map.json` — jurisdiction → county mapping

**What it does:**
Reads each annual CalRecycle disposal tonnage spreadsheet. Columns are Year, Quarter, Jurisdiction, County, Landfill, Transformation, EMSW (Exported Municipal Solid Waste), and Green Material ADC. Computes `total = landfill + transformation + emsw` (Green ADC is excluded per CalRecycle methodology). Aggregates records from all 7 files into a single JSON keyed by jurisdiction name, each value being an array of quarterly records sorted by year and quarter.

---

### `tools/data_transformation/transform_population.py`

**Input:** `data/raw/population/e4-2021-2025.xlsx`

**Output:** `data/processed/population.json`

**What it does:**
Reads the CA DOF E-4 population estimates spreadsheet. Handles the DOF naming convention of appending " City" to disambiguate cities from counties (e.g., "Daly City" is a real name but "Hayward City" should be "Hayward"). A curated allowlist (`REAL_CITY_WITH_CITY`) identifies cases where " City" is genuinely part of the name. An `ALIASES` dictionary handles the handful of cases where the DOF name differs from the CalRecycle canonical name (e.g., "Angels City" → "Angels Camp"). Outputs a JSON keyed by CalRecycle-compatible jurisdiction name.

---

### `tools/data_transformation/transform_characterization.py`

**Input:** `data/raw/waste-characterization/{county}/{stream}/{jurisdiction}_{stream}.xlsx`

**Output:** `data/processed/characterization/{slug}.json` (one file per jurisdiction)

**What it does:**
Walks the entire `data/raw/waste-characterization/` directory tree. For each jurisdiction, reads both the commercial (material type streams) and residential (streams) Excel exports. Aggregates material tons by category using the eight canonical category groups (Organic, Paper & Cardboard, Plastic, etc.) and computes the percentage of total for each. Writes one JSON file per jurisdiction using the slug derived from the jurisdiction name. Jurisdictions without both commercial and residential data get a partial output.

---

### `tools/data_transformation/build_processed.py`

**Input:** All three processed outputs (disposal, population, characterization)

**Output:** `data/processed/jurisdictions.json`

**What it does:**
Joins disposal, population, and characterization data into a single master registry. For each jurisdiction that has Q1 2024 disposal data:
1. Looks up 2024 population from `population.json` (exact name match, then case-insensitive, then alias override via `JURS_TO_POP_ALIASES`)
2. Computes `perCapita` = Q1 2024 total tons × 2000 lbs/ton ÷ population ÷ 90 days
3. Computes `yoy` = (Q1 2024 total − Q1 2023 total) / Q1 2023 total × 100
4. Sets `hasCharacterization = true` if a characterization JSON exists for this jurisdiction
5. Skips jurisdictions with no Q1 2024 disposal record

Run this script last, after all three transform scripts have completed.

---

### `tools/data_transformation/build_city_color_map.py`

**Input:** `data/processed/jurisdictions.json`

**Output:**
- `src/styles/cityPalette.js` (in the wastedata app repo)
- `src/data/cityColorMap.js` (in the wastedata app repo)

**What it does:**
Generates a deterministic, visually-distinct color assignment for all ~419 jurisdictions. Uses a 20-hue palette with varied saturation and lightness to ensure neighboring cities get distinct colors. The algorithm distributes jurisdictions evenly across hues so the full bar chart remains legible. Output files are static artifacts committed to the wastedata repo — re-run whenever new jurisdictions are added to `jurisdictions.json`.

**Note:** This script writes to the wastedata app repo, not this one. Run it from the wastedata-ca-data root and ensure the wastedata repo is at the expected relative path, or update the output paths at the top of the script.

## GitHub Actions Workflows

### `deploy-pages.yml` — Publish data to GitHub Pages

**Trigger:** Push to `main` that modifies any file in `data/processed/characterization/`, `data/processed/jurisdictions.json`, `data/processed/population.json`, or `data/processed/disposal/`. Also triggerable manually via workflow_dispatch.

**What it does:**
Stages a `_site/` directory with two sets of files:
1. All characterization JSON files copied to the root (preserving the existing URL structure at `/{slug}.json`)
2. Core files (jurisdictions, population, disposal) copied into a `core/` subdirectory

Deploys `_site/` to the `gh-pages` branch using `JamesIves/github-pages-deploy-action`. GitHub Pages serves this branch at `https://maxnelson.github.io/wastedata-ca-data`.

### `release-data.yml` — Package core data as a GitHub Release

**Trigger:** Push to `main` that modifies `data/processed/jurisdictions.json`, `data/processed/population.json`, or any file under `data/processed/disposal/`. Also triggerable manually.

**What it does:**
Tarballs the four core files (jurisdictions, population, by_jurisdiction, jurisdiction_county_map) into `core-data.tar.gz` and creates or updates a GitHub Release tagged `data-latest`. This provides a single-download option for consumers who want the full core dataset.

## Updating the Data

When CalRecycle publishes new disposal tonnage data or new characterization study results:

1. **Download updated raw files**
   - For disposal tonnage: download the new annual spreadsheet from CalRecycle and save it to `data/raw/disposal-tonnage/calrecycle-disposal-{year}.xlsx`
   - For population: download the latest E-4 file from CA DOF and replace `data/raw/population/e4-2021-2025.xlsx`
   - For characterization: run the extraction scripts (see Data Extraction above)

2. **Run the transform pipeline**
   ```bash
   npm run transform:all
   ```

3. **Commit and push**
   ```bash
   git add data/processed/
   git commit -m "Update processed data — {month} {year}"
   git push
   ```

4. **GitHub Actions deploys automatically**
   - `deploy-pages.yml` publishes updated files to GitHub Pages within ~1 minute
   - `release-data.yml` creates an updated `data-latest` release
